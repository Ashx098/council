## Components

### 1. Hub (packages/hub)

The central server that stores events, generates digests, and coordinates between components.

**Technology Stack:**
- FastAPI (async web framework)
- SQLite (database)
- Filesystem (artifact storage)

**Key Modules:**

| Module | Purpose |
|--------|---------|
| `main.py` | FastAPI app, endpoints |
| `config.py` | Settings, constants |
| `db/repo.py` | Database CRUD operations |
| `db/schema.sql` | SQLite schema |
| `core/ingest.py` | Event/artifact ingestion |
| `core/digest.py` | Bounded digest generation |
| `core/stream.py` | SSE manager |
| `core/pairing.py` | Pairing service |
| `storage/artifacts.py` | File storage |

### 2. CLI (packages/cli)

Command-line interface for running agents and interacting with the hub.

**Technology Stack:**
- Typer (CLI framework)
- httpx (HTTP client)
- Rich (terminal output)

**Commands:**

| Command | Purpose |
|---------|---------|
| `pair` | Manage pairing codes |
| `attach` | Run agent with output capture |
| `run` | Execute single command |
| `tail` | View recent events |
| `snapshot` | View session state |

**Wrapper Components:**

| Component | Purpose |
|-----------|---------|
| `Runner` | Execute commands, capture output |
| `GitWatcher` | Monitor git diffs |
| `Safety` | Command allowlist |
| `OutputCapture` | Batch output |

### 3. Extension (packages/extension)

Chrome extension for ChatGPT integration.

**Technology Stack:**
- Manifest V3
- Service Worker
- Content Scripts

**Components:**

| Component | Purpose |
|-----------|---------|
| `background.js` | Service worker, SSE client, API calls |
| `content_script.js` | UI injection, toolbar, message handling |
| `content_styles.css` | Styling |
| `ui/panel.html` | Settings popup |
| `lib/dom_extract.js` | DOM utilities |
| `lib/cursor_store.js` | Cursor storage |
| `lib/thread_id.js` | Thread ID extraction |

## Data Model

### Database Schema

```sql
-- Sessions (ChatGPT threads)
CREATE TABLE sessions (
    session_id TEXT PRIMARY KEY,
    repo_root TEXT,
    title TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    event_count INTEGER DEFAULT 0
);

-- Events (messages, patches, etc.)
CREATE TABLE events (
    event_id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT NOT NULL,
    ts TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    source TEXT NOT NULL,
    type TEXT NOT NULL,
    body TEXT,
    meta_json TEXT,
    FOREIGN KEY (session_id) REFERENCES sessions(session_id) ON DELETE CASCADE
);

-- Artifacts (large content)
CREATE TABLE artifacts (
    artifact_id TEXT PRIMARY KEY,
    session_id TEXT NOT NULL,
    kind TEXT NOT NULL,
    path TEXT NOT NULL,
    byte_size INTEGER,
    sha256 TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (session_id) REFERENCES sessions(session_id) ON DELETE CASCADE
);

-- Pairing codes
CREATE TABLE pairing_codes (
    code TEXT PRIMARY KEY,
    session_id TEXT NOT NULL,
    repo_root TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expires_at TIMESTAMP NOT NULL,
    claimed_at TIMESTAMP,
    claimed_by TEXT,
    FOREIGN KEY (session_id) REFERENCES sessions(session_id) ON DELETE CASCADE
);
```

### Session ID Format

```
cgpt:<thread_id>

Examples:
- cgpt:abc123-def456-ghi789
- cgpt:0123456789abcdef
```

### Event ID

Monotonically increasing integer (AUTOINCREMENT) for cursoring.

### MV3 Reliability (Phase 4.1)

1. **chrome.alarms** - Wake every 1 minute
2. **Connection state tracking** - Know when disconnected
3. **Auto-reconnect** - Reconnect on wake or tab focus
4. **Stale detection** - Alert if no events for 3 minutes

### Design Principles

1. **Local-first**: All data stays on your machine
2. **Explicit sync**: User must click Sync/Pull
3. **No auto-send**: Extension never sends to ChatGPT automatically
4. **Bounded output**: Digests are truncated to prevent data leaks

---

## Future Phases

### Phase 6: LLM Summarization

- Replace deterministic truncation with LLM summarization
- Configurable summary styles
- Multi-level compression

### Phase 7: Multi-Repo Support

- Multiple repos per session
- Cross-repo patches
- Unified digest
